# 새 강좌 만들기 가이드

## 이 문서의 목적
이 문서는 `lesson-skill`에 새로운 강좌를 추가하려는 사람을 위한 실전 안내서다.
목표는 화려한 커리큘럼이 아니라,
`/lesson`이 바로 읽고 진행할 수 있는 **일관된 강좌 구조**를 만드는 것이다.

## 기본 폴더 구조
새 강좌는 아래 구조를 따른다.

```text
courses/{course-id}/
├── CLAUDE.md
├── course-structure.json
└── lessons/
    ├── 1-1-topic.md
    ├── 1-2-topic.md
    └── 2-1-topic.md
```

## 파일별 역할
### `CLAUDE.md`
강좌의 세계관과 진행 원칙을 담는다.
다음을 포함하면 좋다.
- 학습 시나리오
- 현재 미션
- 성공 기준
- 학습자 프로필
- 진행 원칙
- 금지된 기대
- 코스의 마지막 장면

핵심은 "이 강좌가 누구를 어디까지 데려가는가"를 분명히 하는 것이다.

### `course-structure.json`
강좌의 메타데이터와 모듈 구조를 정의한다.
현재 표준 스키마는 아래와 같다.

```json
{
  "id": "skill-builder",
  "name": "스킬 빌더 — 나만의 Claude Code 스킬 만들기",
  "level": "intermediate",
  "prerequisite": "cc101",
  "modules": [
    {"id": "1", "name": "모듈명", "lessons": ["1-1", "1-2"]}
  ]
}
```

### `lessons/*.md`
실제 수업에서 읽히는 콘텐츠다.
학습자는 이 문서를 따라 개념을 이해하고 실습 흐름을 잡는다.

## 레슨 작성 템플릿
아래 템플릿으로 시작하면 된다.

```markdown
# 1-1 레슨 제목

## 이 레슨의 위치
이전 레슨과의 연결, 현재 단계의 의미

## 학습 목표
- 목표 1
- 목표 2

## 핵심 개념 또는 실습 시나리오
설명

## 실습
### 실습 1
...

## 체크리스트
- [ ] 항목 1
- [ ] 항목 2

## 한 줄 정리
...

## 다음 레슨
...
```

## 난이도 연결 방법
새 강좌는 단독 섬처럼 만들지 않는다.
반드시 이전 단계와 다음 단계를 연결한다.

- beginner: 처음 시작하는 사람
- intermediate: 기본 사용 경험이 있는 사람
- advanced: 구조와 운영을 설계하는 사람

## 작성 원칙
- 한국어로 쓴다.
- 코드와 명령어만 영문으로 유지한다.
- 한 레슨에는 하나의 중심 개념만 둔다.
- 실습은 "직접 해볼 수 있는 수준"으로 설계한다.
- lesson-sungwoo 같은 다른 프로젝트 문맥과 섞지 않는다.

## 최종 점검
새 강좌를 만들고 나면 아래를 확인한다.
- `CLAUDE.md`만 읽어도 학습자 상황이 보이는가?
- `course-structure.json`이 모듈/레슨 흐름을 명확히 보여주는가?
- 레슨 제목만 읽어도 성장 경로가 보이는가?
- README에 연결했는가?

## 한 줄 정리
좋은 강좌는 내용이 많은 강좌가 아니라,
**누가 왜 배우는지와 어떻게 성장하는지가 구조로 보이는 강좌**다.
