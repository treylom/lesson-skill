# 난이도 로드맵

## 이 문서의 목적
`lesson-skill`의 강좌를 beginner → intermediate → advanced 흐름으로 이해하고,
학습자가 지금 어디쯤 있는지 스스로 진단할 수 있도록 돕는다.

## 전체 흐름
```text
CC101 → skill-builder → context-engineering
```

## 1단계: beginner — CC101
### 이런 사람에게 맞다
- Claude Code를 처음 쓴다.
- 터미널과 Git이 낯설다.
- AI로 실제 업무를 한 사이클 끝내본 적이 없다.

### 여기서 배우는 것
- 파일 탐색
- 데이터/문서 읽기
- 계획 세우기
- 보고서 초안 만들기
- Git/GitHub 첫 사용

### 다음 단계로 갈 준비가 된 신호
- Claude Code에게 일을 순서대로 부탁할 수 있다.
- 결과를 검토하고 수정 요청을 할 수 있다.
- 작은 업무 결과물을 혼자 완성해본 적이 있다.

## 2단계: intermediate — skill-builder
### 이런 사람에게 맞다
- Claude Code 기본 사용은 해봤다.
- 반복 업무를 매번 다시 설명하는 것이 비효율적이라고 느낀다.
- 스킬 구조를 배우고 싶다.

### 여기서 배우는 것
- `SKILL.md` 구조
- frontmatter 설계
- `references/` 분리
- Progressive Disclosure
- 플러그인 메타데이터와 배포 흐름

### 다음 단계로 갈 준비가 된 신호
- 작은 스킬을 직접 설계할 수 있다.
- 입력/출력/흐름을 먼저 정의할 수 있다.
- 긴 스킬을 나눠 관리할 수 있다.

## 3단계: advanced — context-engineering
### 이런 사람에게 맞다
- 스킬을 하나 이상 직접 설계해봤다.
- 이제는 개인 생산성보다 팀 운영 방식이 더 중요하다.
- 에이전트 팀, MCP, 운영 원칙 설계에 관심이 있다.

### 여기서 배우는 것
- 프롬프트, `CLAUDE.md`, Plan 모드의 차이
- 에이전트 팀 운영과 병렬 작업 구조
- MCP 서버와 외부 시스템 연동
- 팀 전체 AI 워크플로우 설계

## 자가 진단 질문
아래 질문에 답해보면 현재 단계가 보인다.

### CC101부터 시작하는 편이 좋은 경우
- Claude Code에게 무엇을 부탁할 수 있는지 아직 감이 없다.
- 파일, Git, 명령어가 아직 부담스럽다.
- 자동화보다 먼저 "한 번 끝까지 해보는 경험"이 필요하다.

### skill-builder가 맞는 경우
- 기본 사용은 익숙하다.
- 반복 업무를 묶어 재사용하고 싶다.
- 좋은 스킬 구조가 궁금하다.

### context-engineering이 맞는 경우
- 개인용 스킬을 넘어 팀 표준을 고민하고 있다.
- 에이전트/도구/MCP를 어디에 써야 할지 설계하고 싶다.
- 같은 질문에 다른 결과가 나오는 이유를 구조적으로 이해하고 싶다.

## 추천 학습 순서
1. CC101으로 실제 업무 한 사이클을 끝낸다.
2. skill-builder에서 반복 업무를 스킬로 묶는다.
3. context-engineering에서 팀 단위 운영 구조를 설계한다.

## 한 줄 정리
난이도는 지식량이 아니라,
**"내가 지금 해결하려는 문제의 범위"** 로 구분하면 가장 정확하다.
