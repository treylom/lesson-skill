# 1-2: "이 폴더에 뭐가 있어?" + practice-data 분석
## 학습 목표
- Claude Code에게 폴더 탐색을 부탁하는 법을 익힌다.
- 파일 이름만 보는 것이 아니라, 무엇부터 읽어야 하는지 판단받는 법을 배운다.
- `practice-data/` 폴더를 중심으로 첫 데이터 탐색을 해본다.
- 매출 데이터와 회의록을 읽기 전, 전체 지도를 만드는 습관을 익힌다.

## 사전 준비
- 1-1 레슨을 마친 상태
- `practice-data/` 폴더가 준비된 코스 디렉토리
- Claude Code 실행 가능 상태
- 파일을 아직 열어보지 않았어도 괜찮다는 마음
- 분석보다 탐색이 먼저라는 이해

## 진행
### 도입 (5분)
- 초보자가 흔히 하는 실수는 파일을 무작정 하나씩 여는 것이다.
- 하지만 실무에서는 먼저 "지형도"를 봐야 한다.
- 폴더 탐색은 서랍장을 여는 일이 아니라, 방 구조를 먼저 보는 일에 가깝다.
- Claude Code는 이때 매우 유용하다.
- "이 안에 뭐가 있지?"
- "어떤 파일부터 보는 게 좋지?"
- "이 파일들은 서로 어떤 관계지?"
- 이런 질문을 자연어로 할 수 있기 때문이다.
- 오늘 김신입은 매출 데이터와 회의록이 있는 폴더를 처음 맡는다.
- 목표는 상세 분석이 아니다.
- 목표는 **파일 구조를 이해하고 읽는 순서를 정하는 것**이다.

#### 오늘의 비유
- 폴더 탐색은 처음 가는 공장에서 동선을 파악하는 것과 같다.
- 생산 라인, 창고, 사무실 위치를 먼저 알아야 일한다.
- 파일도 마찬가지다.

#### 체크 포인트
- 수강생이 `practice-data/` 경로를 인식하는가?
- CSV와 Markdown의 차이를 아주 대략적으로 아는가?
- 지금은 분석보다 개요 파악 단계임을 이해하는가?

### 실습 1 (15분)
#### 실습 목표
- Claude Code에게 폴더 전체를 훑어달라고 요청한다.
- 파일별 역할을 쉬운 말로 요약받는다.
- 어떤 파일부터 읽을지 우선순위를 정한다.

#### 따라 하기
1. Claude Code를 열고 아래처럼 입력한다.

```text
이 폴더 구조를 비개발자 눈높이로 설명해주세요. 특히 practice-data 폴더 안 파일들이 각각 어떤 역할인지 알려주세요.
```

2. 답변이 오면 이어서 이렇게 묻는다.

```text
제가 첫 분석 보고서를 만들어야 한다면 어떤 파일부터 읽는 게 좋을지 순서를 추천해주세요.
```

3. Claude Code가 제안한 순서를 메모한다.
4. 일반적으로는 폴더 설명 파일 → 데이터 파일 → 회의록 순서가 이해하기 쉽다.
5. 이해가 안 되는 파일 형식이 있으면 바로 묻는다.

```text
CSV 파일이 무엇인지, 엑셀 파일과 어떻게 다른지 3줄로 설명해주세요.
```

#### 기대 결과
- `README.md`는 폴더 안내서,
- `sales_data.csv`는 숫자 데이터,
- `meeting_notes.md`는 맥락 자료라는 감을 잡는다.
- 파일을 열기 전부터 머릿속 순서가 생긴다.

#### 실습 메모
- 여기서 중요한 것은 정확한 해석보다 구조 이해다.
- 분석 전에 구조를 잡아두면 나중에 덜 헤맨다.

#### 강사 설명 포인트
- "파일을 잘 읽는 사람은 먼저 질문을 잘 던집니다."
- "무작정 열지 말고, Claude Code에게 길잡이 역할을 먼저 맡기세요."

### 실습 2 (15분)
#### 실습 목표
- `practice-data/sales_data.csv`와 `practice-data/meeting_notes.md`의 핵심만 빠르게 파악한다.
- 자세히 읽기 전에 요약과 관찰 포인트를 받는다.
- 어떤 추가 질문을 던져야 좋은 분석이 되는지 체감한다.

#### 따라 하기
1. 먼저 데이터 파일에 대해 묻는다.

```text
practice-data/sales_data.csv를 읽고, 어떤 열이 있는지와 이 데이터로 알 수 있는 질문 5가지를 알려주세요.
```

2. 답변을 읽고, 바로 더 좁혀 묻는다.

```text
이 데이터에서 초보자가 가장 먼저 볼 만한 패턴 3가지를 추천해주세요.
```

3. 이어서 회의록에 대해 묻는다.

```text
practice-data/meeting_notes.md를 읽고 반복해서 등장하는 문제, 우려, 기회를 정리해주세요.
```

4. 마지막으로 두 파일을 연결한다.

```text
매출 데이터와 회의록을 함께 보면 어떤 가설을 세울 수 있을지 초안 형태로 알려주세요.
```

#### 기대 결과
- 숫자 데이터에서는 계절성, 지역별 차이, 제품별 편차 같은 관찰 포인트가 나온다.
- 회의록에서는 재고, 납기, 영업 이슈 같은 맥락이 드러난다.
- 두 자료를 연결하면 "왜 이런 숫자가 나왔을까"라는 해석이 가능해진다.

#### 좋은 후속 질문 예시
- "남부 지역 매출이 낮은 이유를 회의록 맥락과 연결해 설명해주세요."
- "제품별 매출 차이를 한 문단으로 요약해주세요."
- "보고서 제목 후보 5개를 만들어주세요."
- "임원 보고용과 실무자 공유용을 나눠서 핵심을 정리해주세요."

#### 나쁜 질문 예시와 개선
- 나쁜 질문: "분석해줘"
- 개선 질문: "제품별·지역별·월별로 나눠서 가장 눈에 띄는 변화 3가지를 설명해주세요"
- 나쁜 질문: "회의록 요약해줘"
- 개선 질문: "회의록에서 매출에 영향을 줄 수 있는 운영 문제만 추려서 요약해주세요"

### 마무리 (5분)
- 오늘은 폴더를 읽는 방법을 배웠다.
- 중요한 것은 파일을 많이 여는 것이 아니라, 읽는 순서를 설계하는 것이다.
- Claude Code는 파일 자체만 보는 도구가 아니라, **무엇부터 보면 좋을지 추천하는 안내자**다.
- 김신입은 이제 데이터와 문서 사이를 왔다 갔다 하며 질문할 준비가 되었다.
- 다음 시간에는 `/init`으로 CLAUDE.md를 만들며, AI에게 업무 매뉴얼을 주는 법을 배운다.

#### 오늘의 한 줄 정리
- "좋은 탐색은 좋은 순서에서 시작된다."

#### 실습 후 바로 해볼 것
1. Claude Code에게 "이 폴더에서 초보자가 놓치기 쉬운 파일이 있나요?"라고 묻는다.
2. "제가 보고서 작성자라면 어떤 질문 순서로 분석하면 좋을까요?"라고 묻는다.
3. 답변을 3단계 체크리스트로 바꿔달라고 한다.

## 트러블슈팅
### 증상 1: 파일이 많아 보이거나 복잡해 보인다
- 해결: 전체 설명 대신 아래처럼 범위를 줄인다.

```text
practice-data 폴더만 대상으로 설명해주세요.
```

### 증상 2: CSV가 낯설다
- 해결: 아래처럼 엑셀과 비교해서 설명받는다.

```text
CSV를 엑셀 사용자 관점에서 설명해주세요.
```

### 증상 3: 요약이 너무 추상적이다
- 해결: 기준을 지정한다.

```text
매출 보고서 작성자 관점에서만 요약해주세요.
```

### 증상 4: 회의록과 데이터가 연결되지 않는다
- 해결: 연결 질문을 따로 던진다.

```text
회의록의 어떤 내용이 sales_data.csv 변화와 이어질 수 있는지 가설 형태로 정리해주세요.
```

### 증상 5: 내가 뭘 먼저 해야 할지 다시 헷갈린다
- 해결: 아래처럼 순서를 다시 요청한다.

```text
초보자용 20분 탐색 순서를 다시 짜주세요.
```
