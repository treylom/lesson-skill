# 1-3: /init으로 CLAUDE.md 만들기 "AI에게 업무 매뉴얼"
## 학습 목표
- `CLAUDE.md`가 무엇인지 비개발자 기준으로 이해한다.
- `/init`을 통해 프로젝트용 안내 문서를 만드는 흐름을 익힌다.
- Claude Code가 맥락을 더 잘 이해하도록 배경 정보를 정리하는 법을 배운다.
- 김신입의 업무 환경을 AI에게 알려주는 실습을 한다.

## 사전 준비
- 1-2 레슨까지 완료
- 현재 작업 폴더에 접근 가능
- `practice-data/` 폴더 내용이 대략 이해된 상태
- "AI도 업무 인수인계가 필요하다"는 비유를 받아들일 준비
- 수강생 본인의 역할과 목표를 한두 문장으로 말할 수 있는 상태

## 진행
### 도입 (5분)
- 같은 질문을 해도 맥락이 있으면 답이 달라진다.
- 사람에게도 인수인계 문서가 필요하듯, AI에게도 프로젝트 설명서가 있으면 좋다.
- Claude Code에서 그 역할을 하는 대표 문서가 `CLAUDE.md`다.
- 이 파일은 쉽게 말해 **AI에게 주는 업무 매뉴얼**이다.
- 여기에 "나는 누구인지", "이 폴더는 무엇인지", "무엇을 중요하게 보는지"를 적어둘 수 있다.
- 그러면 매번 긴 설명을 반복하지 않아도 된다.
- 오늘 김신입은 `/init`으로 첫 프로젝트 안내서를 만든다.
- 목표는 완벽한 문서가 아니라, AI가 덜 헤매게 하는 최소한의 맥락을 주는 것이다.

#### 쉬운 비유
- `CLAUDE.md`는 새 팀원에게 건네는 첫 업무 노트다.
- "우리는 이런 팀이고, 이 폴더는 이런 용도고, 지금 목표는 이것입니다"를 적어놓는 문서다.

#### 체크 포인트
- 수강생이 문서 기반 협업의 필요성을 이해하는가?
- 매번 같은 설명을 반복하는 불편을 느껴본 적이 있는가?
- 자기 역할과 목표를 문장으로 표현할 수 있는가?

### 실습 1 (15분)
#### 실습 목표
- `/init`을 실행해 `CLAUDE.md`를 만든다.
- 생성된 내용을 읽고 어떤 정보가 들어가는지 확인한다.
- 필요하면 초보자용 문장으로 다듬는다.

#### 따라 하기
1. Claude Code 입력창에서 아래처럼 입력한다.

```text
/init
```

2. Claude Code가 현재 폴더를 바탕으로 `CLAUDE.md` 생성을 도와주면 내용을 읽는다.
3. 문서가 만들어졌다면 바로 다음 질문을 한다.

```text
방금 만든 CLAUDE.md를 비개발자 관점에서 설명해주세요. 각 섹션이 왜 필요한지도 알려주세요.
```

4. 문장이 너무 기술적이면 아래처럼 요청한다.

```text
지금 문서를 신입 AI 담당자 김신입이 이해할 수 있게 더 쉬운 한국어로 다듬어주세요.
```

5. 문서에 빠진 정보가 있는지 점검한다.

```text
이 프로젝트에서 AI가 더 잘 일하려면 CLAUDE.md에 추가하면 좋은 정보를 5개 추천해주세요.
```

#### 기대 결과
- `CLAUDE.md`는 단순 메모가 아니라 AI 협업용 컨텍스트 문서라는 점을 이해한다.
- 생성된 내용을 무조건 믿는 대신, 사람이 검토하고 수정해야 함을 배운다.
- 수강생은 "문서로 AI를 훈련시킨다"는 감각을 얻는다.

#### 강사 설명 포인트
- "좋은 CLAUDE.md는 긴 문서가 아니라 도움이 되는 문서입니다."
- "자주 반복하는 설명, 중요한 목표, 금지사항을 적는 것이 핵심입니다."

### 실습 2 (15분)
#### 실습 목표
- 김신입 시나리오에 맞게 `CLAUDE.md` 내용을 구체화한다.
- 프로젝트 목표, 데이터 설명, 성공 기준을 문서에 반영한다.
- 이후 레슨에서 계속 재사용할 기반을 만든다.

#### 따라 하기
1. 아래 문장을 그대로 입력한다.

```text
CLAUDE.md에 다음 맥락이 반영되어 있는지 확인해주세요. 저는 제조업 중견기업의 신입 AI 담당자 김신입이고, practice-data의 매출 데이터와 회의록으로 첫 분석 보고서를 만들고, 결과를 GitHub 저장소로 정리하는 것이 목표예요. 부족한 내용을 제안해주세요.
```

2. Claude Code가 제안한 수정안을 읽는다.
3. 이어서 아래처럼 요청한다.

```text
CLAUDE.md에 들어갈 내용으로, 역할 / 현재 미션 / 성공 기준 / 데이터 설명 / 작업 원칙 섹션 초안을 작성해주세요.
```

4. 초안이 마음에 들면 필요한 표현만 손본다.
5. 마지막으로 문서 활용법을 묻는다.

```text
앞으로 제가 Claude Code에게 질문할 때, CLAUDE.md가 있으면 무엇이 더 좋아지는지 예시 3개로 설명해주세요.
```

#### 기대 결과
- 수강생은 프로젝트 설명을 매번 다시 치지 않아도 되는 이유를 이해한다.
- "역할 + 목표 + 데이터 + 성공 기준"의 4요소가 매우 중요하다는 것을 배운다.
- 이후 분석, 계획, 문서 작성 레슨의 품질이 좋아질 준비가 된다.

#### 좋은 내용 예시
- 내 역할: 신입 AI 담당자
- 현재 목표: 첫 분석 보고서 작성
- 참고 데이터: sales_data.csv, meeting_notes.md
- 원하는 말투: 비개발자 눈높이, 단계별 설명
- 금지: 한 번에 너무 많은 명령 제시 금지

#### 피해야 할 내용
- 너무 추상적인 문장만 가득한 경우
- 실제 목표 없이 멋있는 말만 적는 경우
- 본인이 이해하지 못하는 기술 정책을 그대로 복붙하는 경우

### 마무리 (5분)
- 오늘은 AI에게 "우리 팀 사정"을 알려주는 법을 배웠다.
- `CLAUDE.md`는 개발자 전용 문서가 아니다.
- 오히려 비개발자일수록 더 큰 도움을 받을 수 있다.
- 매번 설명을 반복하지 않아도 되고,
- 더 일관된 답을 받을 수 있고,
- AI가 엉뚱한 방향으로 가는 일을 줄일 수 있다.
- 다음 시간에는 Plan 모드를 통해 "설계도 먼저 그리고 실행하기"를 배운다.

#### 오늘의 한 줄 정리
- "좋은 AI 협업은 좋은 맥락 문서에서 시작된다."

#### 실습 후 바로 해볼 것
1. 현재 폴더용 CLAUDE.md에 꼭 들어가야 할 한 줄을 직접 써본다.
2. Claude Code에게 "이 문서가 있으면 어떤 질문이 더 잘 먹히는지" 물어본다.
3. 본인 업무에서도 자주 반복하는 배경 설명이 무엇인지 떠올려본다.

## 트러블슈팅
### 증상 1: `/init`이 낯설고 무섭다
- 해결: 먼저 기능을 설명받는다.

```text
/init이 무슨 역할인지 먼저 설명해주세요.
```

### 증상 2: 생성된 문서가 너무 길다
- 해결: 핵심만 남기게 요청한다.

```text
초보자에게 꼭 필요한 내용만 남겨서 간단 버전으로 줄여주세요.
```

### 증상 3: 어떤 내용을 넣어야 할지 모르겠다
- 해결: 아래 4가지만 먼저 넣는다.
- 나는 누구인가
- 지금 목표는 무엇인가
- 어떤 파일을 주로 다루는가
- 어떤 방식으로 설명받고 싶은가

### 증상 4: AI가 내 상황을 여전히 잘 모르는 것 같다
- 해결: 성공 기준과 금지사항을 더 명확히 적는다.

```text
성공 기준과 피해야 할 답변 방식도 문서에 반영해주세요.
```

### 증상 5: 문서를 만들었지만 언제 쓰이는지 모르겠다
- 해결: 아래처럼 활용 예시를 요청한다.

```text
이 CLAUDE.md를 활용한 질문 예시를 5개 만들어주세요.
```
