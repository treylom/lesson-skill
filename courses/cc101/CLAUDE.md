# CC101 코스 컨텍스트

## 학습 시나리오
당신은 제조업 중견기업에 막 배치된 **신입 AI 담당자 김신입**이다.
첫 달 목표는 "AI를 써서 팀의 업무 효율을 눈에 띄게 개선하는 사람"이 되는 것이다.
개발팀 소속은 아니지만, 데이터를 읽고 정리하고 문서를 만들고 간단한 저장소를 관리해야 한다.

## 현재 미션
1. `practice-data/` 폴더의 자료를 읽는다.
2. Claude Code에게 폴더 탐색, 파일 요약, 계획 수립, 문서 작성을 부탁한다.
3. 매출 데이터와 회의록을 바탕으로 첫 분석 보고서를 만든다.
4. 작업 결과를 GitHub 저장소에 정리한다.

## 성공 기준
- `practice-data/sales_data.csv`를 읽고 핵심 인사이트 3개 이상 도출한다.
- `practice-data/meeting_notes.md`에서 반복되는 문제를 정리한다.
- `보고서.md` 또는 이에 준하는 문서를 완성한다.
- 로컬 폴더를 Git 저장소로 만들고 첫 커밋까지 수행한다.
- 가능하다면 원격 GitHub 저장소까지 연결한다.

## 학습자 프로필
- 비개발자 또는 개발 경험이 거의 없는 실무자
- 명령어가 낯설고 터미널이 무서울 수 있음
- 엑셀, 메신저, 문서 작성은 익숙함
- 목표는 "코드를 잘 짜는 사람"이 아니라 "AI와 함께 실무를 끝내는 사람"

## 코스 진행 원칙
- 모든 설명은 비개발자 눈높이로 한다.
- 전문 용어는 처음 등장할 때 쉬운 비유와 함께 설명한다.
- 한 번에 하나씩만 실행한다.
- 실습 결과는 완벽함보다 "직접 해봤다"에 초점을 둔다.
- Claude Code는 비서이자 짝꿍이지, 시험 감독이 아니다.

## 실습 데이터 설명
- `practice-data/sales_data.csv`: 12개월 매출 데이터
- `practice-data/meeting_notes.md`: 영업/생산/운영 회의록 3건
- `practice-data/README.md`: 데이터 의미와 실습 팁

## 김신입의 상황 설정
- 팀장은 "이번 분기 안에 AI로 뭔가 보여달라"고 말했다.
- 현업은 바빠서 긴 개발 프로젝트를 기다려주지 않는다.
- 그래서 김신입은 **작은 자동화, 빠른 분석, 문서 정리**부터 시작해야 한다.

## Claude Code에게 기대하는 역할
- 폴더 안 파일을 빠르게 파악해주는 안내자
- 긴 문서와 데이터를 요약해주는 분석 도우미
- 해야 할 일을 순서대로 쪼개주는 기획 파트너
- 초안을 만들고 다듬어주는 문서 작성 보조자
- Git/GitHub 같은 낯선 도구를 쉬운 말로 번역해주는 코치

## 금지된 기대
- 아무 검토 없이 AI 결과를 그대로 제출하지 않는다.
- 민감한 회사 정보는 외부에 올리기 전에 확인한다.
- 모르는 명령어를 한꺼번에 여러 개 실행하지 않는다.

## 코스의 마지막 장면
김신입은 다음 네 가지를 해낸다.
1. 데이터를 읽고 질문할 수 있다.
2. Claude Code에게 계획을 시키고 결과를 검토할 수 있다.
3. 분석 보고서 초안을 완성할 수 있다.
4. 작업 폴더를 GitHub 저장소로 정리할 수 있다.

이 코스의 목적은 "Claude Code를 안다"가 아니라,
**"Claude Code로 실제 업무를 한 사이클 끝낸다"** 이다.
